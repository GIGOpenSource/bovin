# Bovin 量化面板 · 部署指南

面向 **静态资源 + Bovin REST（及可选本地代理）** 的部署流程。细节与安全条款亦见 [README.md](./README.md)。

---

## 一、本地部署还是服务器部署？

| 维度 | **本地化部署**（本机 / 内网只给少数人） | **服务器部署**（远端主机、可能对多人或公网） |
|------|----------------------------------------|------------------------------------------------|
| **典型场景** | 开发调试；机器人与面板 **同一台电脑**；仅自己或内网运维访问 | 团队共享、远程盯盘、需要固定域名与 **HTTPS** |
| **网络** | `http://127.0.0.1:端口` 或 **内网 IP**，尽量不暴露公网 | 公网或 VPN 后统一入口，前加 **Nginx/Caddy** 等反代 |
| **证书** | 多可继续用 HTTP（仅限本机或隔离内网） | **强烈建议 HTTPS**（避免混合内容、满足合规） |
| **复杂度** | 低：Python 静态服务即可 | 中：反代、TLS、防火墙、日志与备份 |
| **风险** | 低（访问面小）；勿误把本机 HTTP 页嵌到 HTTPS 站点里 | 中高：暴露面大，必须收紧鉴权与 **仅反代可信上游** |

### 建议结论（可直接采纳）

1. **个人使用、机器人跑在自己电脑上** → 优先 **本地化部署**：面板与 API 同源或同可信网段，用系统浏览器打开 **`http://127.0.0.1:3000`**（或你设定的端口），**不要用编辑器内置 HTTPS 预览**（易触发混合内容拦截）。
2. **多人协作、生产环境、需要外网访问** → 采用 **服务器部署**：静态资源与（或）API 由 **HTTPS 反代** 提供；Bovin `api_server` **勿裸奔公网**，应配合 VPN、IP 白名单、mTLS 或仅限内网可达，具体由你们安全基线决定。
3. **同一 LAN 内多台机器**：可在内网服务器上按「服务器部署」Hosting 静态页，但仍 **可不启用公网**，仅内网 DNS/IP 访问——本质介于两者之间，安全模型按「谁能进内网」评估。

## 二、部署前准备（两种方案通用）

### 2.1 构建静态资源（推荐）

在 `frontend/`（仓库根下）目录下：

```bash
npm install
npm run build
```

生成压缩后的 `main.js`，并在 `index.html` 写入 `main.js?v=<哈希>`，利于缓存刷新。  
**部署到服务器时请将整个 `frontend/` 目录中至少包含**：`index.html`、`main.js`、`styles.css`、`vendor/`、`src/locales/` **不参与浏览器运行**（文案已打进 `main.js`），可不拷贝；若你希望服务器上可二次构建，可保留 `src/` 与 `package.json`。

### 2.2 Bovin API

- 确认 `api_server` 已启用，且你知道 **基础地址**，例如：`http://127.0.0.1:18080/api/v1`（端口以实际 `config` 为准）。
- **跨域**：若面板与 API **不同源**，须在 Bovin 配置 `CORS_origins` / `CORS_origin_regex`，或 **统一经同域反代**。

### 2.3 可选：本地聚合代理 `binance_proxy.py`

在 **本机或能与浏览器、Bovin 互通的机器** 上运行（默认 `19090`），常见 Base URL：`http://192.168.77.46:8000/api/v1`。  
作用：合并币安行情与 Bovin 转发，减轻跨端口 CORS 问题。环境变量与默认口令见 [README.md](./README.md)。

## 三、方案 A：本地化部署（推荐入门与单人场景）

### 3.1 启动步骤

在 **freqtrade 仓库根** 或 `frontend` 目录按习惯执行：

```bash
cd frontend
chmod +x start_frontend.sh check_frontend_health.sh
./start_frontend.sh
```

默认 **`http://127.0.0.1:3000`**。仅检查依赖与连通性、不启服务：

```bash
./start_frontend.sh --check-only
```

自定义端口示例：

```bash
FRONTEND_PORT=3001 ./start_frontend.sh
```

**HTTP 与 HTTPS 并行（可选）**：需要「电脑继续用 `http://IP:3000`、手机用 `https://IP:3443`」时，可在 `frontend` 下执行 `./start_frontend_dual.sh`（会先跑健康检查，再同时起 `http.server` 与 `serve_https_lan.py`）。HTTPS 入口自带 API 反代；纯 HTTP 入口仍依赖本机或局域网可达的 `19090` / Bovin，**跨端口时请在 API 侧放行浏览器来源**（`CORS_origins` 等）。

等价手动命令（不跑健康检查）：

```bash
python3 -m http.server 3000 --bind 127.0.0.1 --directory frontend
```
（在仓库根执行时注意 `--directory` 指向 `frontend`。）

### 3.2 面板内配置

1. 浏览器打开上述 URL（地址栏须为 **`http://`**）。  
2. **登录**（面板账号，与 Bovin API 账号可能不同）。  
3. **设置中心** 填写 **Base URL**：  
   - 使用代理：`http://192.168.77.46:8000/api/v1`  
   - 直连 Bovin：`http://192.168.77.46:8000/api/v1`（示例）

### 3.3 注意事项

- 勿用 **`file://`** 打开 `index.html`。  
- 若页面在 **HTTPS** 环境而 API 是 **HTTP**，浏览器会拦截（混合内容）；请改为全程 HTTP 本机，或为 API 配置 **HTTPS 同源反代**。

## 四、方案 B：服务器部署（团队 / 生产向）

### 4.1 总体架构建议

```
用户浏览器 (HTTPS)
        ↓
反向代理 (Nginx / Caddy)：TLS 终止、静态 root、/api/v1 反代到 Bovin
        ↓
静态文件 ← 来自 frontend 构建产物
        ↓（/api/v1）
Bovin（建议仅内网或 VPN 可达，或限制来源 IP）
```

理想情况：**面板静态页与 API 同源**（例如同一域名下 `/` 为静态，`/api/v1` 反代到 Bovin），这样浏览器策略最简单。

### 4.2 上传文件

将 **`frontend/` 部署目录** 至少包含：

- `index.html`
- `main.js`（及当前 `?v=` 指纹，以构建产物为准）
- `styles.css`
- `vendor/lightweight-charts.standalone.production.js`

上传到服务器选定目录，例如 `/var/www/bovin-panel/`。

### 4.3 Nginx 配置示意（仅供参考）

以下仅为 **思路示例**，实际路径、证书、上游地址需替换；生产需由运维复核。

```nginx
server {
    listen 443 ssl http2;
    server_name panel.example.com;

    # ssl_certificate / ssl_certificate_key ...

    root /var/www/bovin-panel;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # 将 API 与前端同源（推荐）
    location /api/v1/ {
        proxy_pass http://127.0.0.1:18080/api/v1/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header Authorization $http_authorization;
    }
}
```

部署后，**设置中心 Base URL 留空或填** `https://panel.example.com/api/v1`（按你们反代路径调整），使浏览器只访问 **HTTPS 同源**。

### 4.4 若静态与 API 不同域

- 必须在 Bovin 侧正确配置 **CORS**，且仍建议 **HTTPS + 谨慎放开 Origin**。  
- 密钥与口令策略见 README「安全与隐私」。

### 4.5 缓存建议

- **HTML**：短缓存或 `no-cache`，避免旧壳引用新脚本失败。  
- **带 `?v=` 的 `main.js`**：可长缓存。  
生产构建已自动为 `main.js` 追加内容哈希查询参数。

## 五、云部署说明（公有云 / CDN / 容器）

本节说明在**阿里云、腾讯云、AWS、GCP**等环境托管**面板静态资源**，并与**云上或混合部署的 Bovin** 对接时的常见做法。原则与上文 **第四节 · 方案 B（服务器部署）** 一致：**浏览器侧尽量 HTTPS 同源 `/api/v1` 反代到 Bovin**，避免混合内容与宽松 CORS。

### 5.1 典型拓扑（三选一或组合）

| 模式 | 静态面板 | Bovin API | 适用 |
|------|----------|-----------|------|
| **A. 单台云主机** | 与 Nginx/Caddy 同机，`root` 指向构建产物 | 同机 `127.0.0.1:18080` 或 Docker 网桥 | 小团队、成本低、运维简单 |
| **B. 对象存储 + CDN + 网关反代** | 桶静态网站/静态托管 + CDN；**API 经 CDN/负载均衡同源反代** | 内网 SLB 后端、或另一组 ECS/容器 | 高并发读静态、全球加速 |
| **C. 静态托管 + API 异源** | 如 OSS 静态网站绑定自定义域 | 另一子域 `api.*` | 必须在 Bovin 配 **CORS**；不如同源省心 |

**推荐**：优先 **A** 或 **B 的同域反代**（用户只访问 `https://panel.example.com`，`/api/v1` 由边缘或源站转发）。

### 5.2 对象存储 + CDN（阿里云 OSS / 腾讯云 COS / AWS S3 等）

1. 本地执行 `npm run build`，将 `index.html`、`main.js`、`styles.css`、`vendor/` 上传至桶（保持相对路径，**勿改** `index.html` 里 `./main.js`、`./styles.css` 引用方式；若 CDN 对根路径有特殊规则需实测）。  
2. **仅托管静态文件不够**：纯静态桶**无法**处理 `POST /api/v1/panel/*`。必须二选一：  
   - **同源反代**：在 **CDN/全站加速**、**API 网关**、或前置 **Nginx/OSS 回源到计算实例** 上，把 `https://你的域名/api/v1/` 转发到 Bovin；或  
   - **异源 API**：面板设置里填 `https://api.example.com/api/v1`，并在 Bovin 配置 **HTTPS + CORS**。  
3. **缓存**：桶/CDN 上对 `index.html` **勿长缓存**；对带 `?v=` 的 `main.js` 可长缓存（与第四节 4.5 一致）。  
4. **证书**：在 CDN 或负载均衡上绑定 **TLS 证书**（云厂商免费证书或 ACME）。

### 5.3 云服务器（ECS / CVM / EC2 / Compute Engine）

1. 创建 VPC、安全组：**入站**放行 `443`（及可选 `80` 跳转 HTTPS）；**勿**对公网暴露 Bovin 原始端口，仅 **本机或内网** 监听。  
2. 安装 Nginx/Caddy，按 **第四节 4.3** 配置 `root` + `location /api/v1/` → `http://127.0.0.1:18080/api/v1/`（或 Docker 内网 IP）。  
3. Bovin 与 `binance_proxy`（若用）可同机 **systemd** 或 **Docker Compose** 运行；确保防火墙仅本机回环或内网互通。  
4. **数据盘**：`user_data`（含 `panel_store.sqlite`）建议挂云盘并做**快照备份**。

### 5.4 容器与 Kubernetes

1. **镜像**：可用「Nginx + 静态文件」镜像复制构建产物；或 Init 容器从 CI 产物解压。  
2. **Ingress**：同域路径 `/api/v1` **Ingress 规则** 指向 Bovin **Service**（ClusterIP），静态路由指向前端 **Service**；避免浏览器跨域。  
3. **密钥**：`api_server` 密码、交易所密钥等用 **Secret** 挂载环境变量或文件，**勿**打进前端镜像。  
4. **健康检查**：探针可打 Bovin `/api/v1/ping`（按需带鉴权策略由平台决定）。

### 5.5 Bovin 在「另一环境」时（混合云 / 仅面板上网）

- 机器人跑在**办公室/机房**，面板在**公有云**：云上前置 Nginx **反代**到 **VPN / 专线 / 零信任隧道** 暴露的上游地址（须稳定、低延迟、且满足安全审计）。  
- **不建议**在公网直接暴露无额外防护的 Bovin 端口；至少 **IP 白名单 + TLS + 强口令**。

### 5.6 云上与「对接交易所」相关的注意

- 浏览器**永远**只连你配置的 **Bovin Base URL**（同源 `https://域名/api/v1` 或显式 API 域）。  
- **ccxt 访问交易所**发生在 **Bovin 进程所在机器**；若交易所需 **出口 IP 白名单**，须填 **该云主机/Egress NAT 的公网 IP**，不是 CDN 边缘 IP。  
- 云上若仍见 **501 + HTML**，说明 POST 未到达 Bovin（常见：CDN 只缓存了静态、未把 `/api/v1` 回源到 API）。详见 [docs/panel/DEPLOYMENT.md](../docs/panel/DEPLOYMENT.md) 与 [RISKS_AND_AUDIT.md](../docs/panel/RISKS_AND_AUDIT.md)。

## 六、部署后验收清单

- [ ] 用 **系统浏览器** 打开页面（非编辑器预览）。  
- [ ] **登录** 成功，切换各个主导航无报错。  
- [ ] **系统概览** 能拉到 `/ping` 或等效健康信息。  
- [ ] **数据面板** 能加载 K 线（视环境需代理或交易所可达）。  
- [ ] **设置保存** 与 Bovin/面板后端一致（按是否启用 SQLite 面板存储）。  
- [ ] 生产环境：**HTTPS**、**防火墙**、**默认口令已修改**、**错误上报 URL 如启用则指向可信域**。
